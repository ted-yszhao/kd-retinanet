\chapter{Methodology}

This chapter introduces the proposed RetinaNet knowledge distillation framework. It first formalizes the training objective, then describes the teacher-student architecture, and finally explains the feature dimensionality choices that make direct structural distillation possible without an additional adaptation layer.

\section{Overview of the Distillation Framework}

Let a teacher detector $T$ and a student detector $S$ operate on an input image $x$ with ground-truth annotations $y$. Both models follow the RetinaNet detection framework, but they may use different ResNet backbones \cite{he2016resnet,lin2017focal}. The goal is to train the student so that it maintains the inference efficiency of a compact detector while recovering part of the representational strength of a larger teacher. During training, the teacher is frozen and serves only as a source of supervisory signals. During inference, only the student is used.

The student is trained with two objectives: the native RetinaNet detection loss and a feature distillation loss \cite{hinton2015distilling,lin2017focal}. Denoting the student detection loss by $\mathcal{L}_{det}$ and the distillation loss by $\mathcal{L}_{kd}$, the total objective is
\begin{equation}
  \mathcal{L} = \mathcal{L}_{det} + \lambda \mathcal{L}_{kd},
  \label{eq:total-loss}
\end{equation}
where $\lambda$ is a scalar hyperparameter controlling the contribution of distillation. In the implementation used in this project, $\lambda=0.5$ by default.

\section{Teacher and Student Architecture}

Both teacher and student are instantiated as RetinaNet detectors implemented with \texttt{torchvision}. This choice keeps the experiment within a single one-stage detection family, so the comparison is not confounded by changes in proposal generation, transformer queries, or other detector-specific mechanisms that would make the effect of distillation harder to isolate \cite{lin2017focal}. In both branches, RetinaNet provides the same dense prediction heads and the same anchor-based detection formulation, while the main architectural difference lies in the capacity of the underlying residual backbone. The teacher uses a stronger ResNet-FPN backbone constructed through \texttt{resnet\_fpn\_backbone}, whereas the student uses either the standard \texttt{retinanet\_resnet50\_fpn} shortcut or a custom RetinaNet built from a lighter ResNet-FPN backbone. This design supports controlled experiments in which the detector head and training framework remain fixed while backbone depth and representational power change \cite{he2016resnet,lin2017fpn,lin2017focal}.

The teacher branch is configured with a stronger backbone such as \texttt{ResNet-101} or \texttt{ResNet-50}, and its parameters are frozen throughout training. This follows the standard teacher-student setting introduced in knowledge distillation, where a larger network serves as a stable source of supervisory structure while only the student is updated \cite{hinton2015distilling}. Architecturally, the teacher begins with the usual ResNet stem and residual stages, whose deeper layers progressively trade spatial detail for semantic abstraction \cite{he2016resnet}. Such backbone stages are powerful but not sufficient on their own for dense detection, because object detection must localize targets across a wide range of scales. RetinaNet therefore augments the backbone with an FPN, which constructs a multi-resolution hierarchy by combining top-down semantic information with lateral connections from intermediate backbone stages \cite{lin2017fpn}. In the implementation used here, the teacher exposes the selected backbone stages, projects them into a common pyramid width, and then extends the hierarchy with extra coarse levels analogous to $P_6$ and $P_7$, just as in standard RetinaNet \cite{lin2017focal}. The result is a frozen teacher whose main contribution is not a final soft label distribution alone, but a collection of semantically aligned multiscale feature targets.

The student branch is fully trainable and optimized with backpropagation, but it preserves the same high-level detection pipeline. This is an important design choice. The student is not treated as a fundamentally different detector; rather, it is a lower-capacity RetinaNet whose lighter backbone, such as \texttt{ResNet-18}, is expected to reduce computational cost while retaining the same dense prediction structure. In practice this means that the student still contains a residual feature extractor, an FPN neck, and RetinaNet classification and regression subnets. Keeping these components fixed makes the architectural relationship between teacher and student much cleaner than in a heterogeneous transfer setting. Prior distillation work has repeatedly shown that intermediate supervision is easier to interpret and often more effective when the student shares meaningful internal structure with the teacher, whether through hidden-layer hints, attention maps, or feature imitation targets \cite{romero2015fitnets,zagoruyko2017paying,chen2017learning,wang2019distilling,yang2022focal}. The present method adopts exactly that principle by aligning the teacher and student within the same detector family and moving the transfer interface to the level where both branches already encode multiscale detection semantics.

This shared interface is the FPN output rather than the raw backbone output or final detector logits. The distinction matters. If the distillation loss were applied directly to backbone stages, the student and teacher would inherit large architectural mismatches from their underlying residual networks. A bottleneck network such as \texttt{ResNet-50} or \texttt{ResNet-101} produces substantially wider late-stage tensors than a basic-block network such as \texttt{ResNet-18}, so direct matching at the backbone level would generally require a learned projection layer before a stable feature loss could be computed. By contrast, once the features have passed through the FPN, both branches have already been transformed into a common multiscale representation whose levels play the same semantic role for dense prediction \cite{lin2017fpn}. This is also a more task-relevant interface than final logits alone: detector-specific distillation methods have shown that useful knowledge in detection is often carried by intermediate features, foreground structure, and spatial relations rather than only by output probabilities \cite{wang2019distilling,dai2021gid,guo2021defeat,kang2021icd,du2021frs,li2022rank,yang2022focal}.

In the current implementation, pyramid features are extracted after the input images are processed by RetinaNet's internal transform and backbone modules. If the backbone returns an ordered dictionary of feature maps, those maps are used directly as distillation targets. This choice defines the exact interface at which teacher and student are compared: the distillation signal is applied to FPN outputs rather than to raw backbone stages or final detection logits. The advantage is that teacher and student are compared in a space that is already normalized by the detector's own architectural logic. Each selected pyramid level corresponds to a fixed position in the RetinaNet hierarchy, carries the same functional role in the downstream detection heads, and has already been projected into the common feature width expected by those heads. As a result, the supervision reflects how the detector actually represents multiscale object evidence, not an arbitrary earlier activation space that may differ significantly between backbones.

This setup also explains why no additional learned adaptation layer is necessary after the FPN. Because teacher and student remain within the same RetinaNet-FPN scaffold, their level-aligned pyramid outputs already share both a semantic interpretation and a common channel dimensionality. Any remaining mismatch is primarily spatial and can therefore be handled by interpolation rather than by adding extra trainable modules between the two branches. Avoiding such an adapter is not merely an implementation convenience. It keeps the distillation path lightweight, avoids introducing extra parameters that might themselves absorb part of the knowledge transfer, and makes the empirical comparison with student-only training easier to interpret. In that sense, the teacher-student architecture is designed not only for performance, but also for methodological clarity: the teacher provides a stronger multiscale representation, the student preserves the efficient deployment target, and the shared RetinaNet-FPN structure creates a clean and defensible interface across which structural feature knowledge can be transferred \cite{he2016resnet,lin2017fpn,lin2017focal,hinton2015distilling,wang2019distilling,derijk2022structural}.

\section{Dimensionality and Feature Alignment}

Let the transformed and padded image batch entering the detector be denoted by
\begin{equation}
  x' \in \mathbb{R}^{B \times 3 \times H \times W},
\end{equation}
where $B$ is the batch size and $H \times W$ is the spatial size after RetinaNet preprocessing. Before the FPN, the teacher and student backbones need not have matching internal dimensionality. A bottleneck backbone such as \texttt{ResNet-50} or \texttt{ResNet-101} produces deeper late-stage tensors than a basic-block backbone such as \texttt{ResNet-18}. In practical terms, if one considers the common high-level backbone stages $C3$, $C4$, and $C5$, a bottleneck network exposes channel widths of approximately $(512,1024,2048)$, whereas a lighter \texttt{ResNet-18} exposes approximately $(128,256,512)$. For this reason, direct feature matching on raw backbone outputs would generally require an explicit projection layer, typically a learned $1 \times 1$ convolution, to align student and teacher channels before a loss could be computed reliably.

The proposed method avoids this mismatch by distilling after the FPN rather than before it. The FPN already serves as a built-in dimensionality unifier: each selected backbone stage is projected into a common feature width and fused into a pyramid representation with fixed channel dimension. As a result, for every selected pyramid level $l \in \mathcal{P}$,
\begin{equation}
  F_s^{(l)},\; F_t^{(l)} \in \mathbb{R}^{B \times C \times H_l \times W_l},
\end{equation}
with a shared pyramid width $C=256$ in the standard RetinaNet configuration. The spatial dimensions $H_l$ and $W_l$ depend on the pyramid level rather than on the backbone identity. For the usual RetinaNet levels $P3$ through $P7$, the corresponding strides with respect to the transformed input are $8,16,32,64$, and $128$ pixels, so the feature resolutions decrease systematically as depth increases.

This post-FPN alignment is the main reason no additional adaptation layer is needed between student and teacher features in the present design. The teacher and student already expose tensors with the same semantic role and the same channel dimensionality at each aligned pyramid level. Therefore, the distillation loss can compare level-matched features directly. When a small spatial discrepancy remains because of implementation details or image padding, simple bilinear interpolation is sufficient to align $H_l$ and $W_l$ before computing the loss. No learned adapter is required because the mismatch is not one of feature width or representation type, but only of sampling resolution. Avoiding a trainable adaptation layer also keeps the distillation path lightweight, prevents extra parameters from absorbing part of the knowledge transfer, and makes the empirical comparison with student-only training easier to interpret.
